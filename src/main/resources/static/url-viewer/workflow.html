URLViewer 업무 흐름 개요
🏆 IT개발실/IT카드개발팀(백명선, 김수현, 박귀렬 등)이 전체 프로세스를 주도하며, 분석 → 차단 확정 → 배포 → 모니터링까지 책임감을 갖고 이끌어 갑니다. 이 흐름은 1~3개월마다 반복 수행됩니다.
URLViewer는 시스템에 등록된 URL을 자동으로 수집·분석하고, 실제 사용 여부와 변경 이력을 종합하여 차단 대상 여부를 판정하는 내부 관리 도구입니다. 아래 흐름은 관리자(IT카드개발팀)가 수집·설정하고, 자동 처리가 주기적으로 데이터를 갱신하며, 판정 결과를 현업 검토자가 최종 확인하는 전체 업무 흐름을 보여줍니다.

🔵 보류②-① 호출0건+로그건status='②-① 호출0건+로그건'

②-④ 호출 reviewThreshold+1건↑status='②-④ 호출 reviewThreshold+1건↑'
사용으로 변경status='검토필요 → 사용' (수동)
설정값: reviewThreshold(기본 3) — ②-③/②-④ 분기 기준, reviewUpperThreshold(기본 10) — ②-④의 호출 상한. recentLogOnly — 1년 미만 커밋이 모두 "로그/불필요" 키워드 포함 시 true (자동 갱신).
⚙️ 자동 재계산 sticky 규칙 (umbrella 보존)
calculateStatus 는 statusOverridden=false 인 레코드에만 동작합니다.
강한 신호 (무조건 우선):
  1. MANUAL_STATUSES (차단대상 → 사용 / 검토필요 → 사용) → 보존
  2. hasUrlBlock=Y → ①-① 차단완료
  3. reviewResult='차단대상 제외' → ①-⑤ 현업요청 차단제외
Umbrella sticky: 위 강한 신호가 없을 때만 적용
  • 현재 사용 또는 ①-① 차단완료: 조건에 따라 leaf 자유 할당
  • 현재 ①-* / ②-* leaf 내부: target=USE 일 때만 '사용' 으로 전이, 그 외엔 leaf 보존 (차단↔추가검토 전이 / leaf 변경 금지)
효과: 한 번 분류된 레코드의 leaf 는 호출수·커밋·플래그 변동에도 안정적으로 유지. 새 변경 반영하려면 사용자가 수동으로 사용 으로 되돌린 뒤 다음 분석에서 재분류 유도.

분석 · 검토 단계
👤
IT개발실
STEP 1 · 차단 대상 확정
URL 분석현황 화면에서 차단 대상 URL을 필터링하고 상태를 확정합니다.
차단 대상 목록을 엑셀로 다운로드하여 현업 검토 요청용 자료로 활용합니다.
↓
📋
현업검토
STEP 2 · 현업 검토 요청 / 회신
① IT개발실이 협조전을 상신하여 공식 검토 요청을 진행합니다.
② 엑셀(URL 목록)을 첨부하여 메일로 검토 요청을 추가 발송합니다.
③ 차단확정 대상은 SmartWay 티켓을 통해서도 현업 담당자에게 검토를 요청합니다.
현업 담당자 회신: 차단대상 제외 / 차단확정 / 판단불가
↓
👤
IT개발실
STEP 3 · 검토결과 엑셀 업로드

검토결과가 일괄 반영되어 각 URL의 상태에 적용됩니다.
↓

STEP 6 · 개발 및 테스트
차단 확정된 URL은 메서드 본문에 차단 throw 를 배치하고 (위치 무관, 메시지에 "차단" 포함), 폐기 표시·근거 주석을 함께 반영합니다.
① UnsupportedOperationException throw 필수 · ①-① 차단완료 판정 기준 ② @Deprecated 어노테이션 (폐기 표시) ③ [URL차단작업] javadoc 주석 (차단 근거·일자·CSR)
📝 ②·③ 중 누락 시 차단처리미흡 부가 표기 (상태는 ①-① 차단완료 유지)
소스 수정 완료 후 단위 테스트 및 통합 검증을 수행합니다. 차단 URL 호출 시 정상 차단되는지, 기존 기능에 영향은 없는지 확인합니다.

9. URL 차단 여부 판정 기준
소스 수집 시 각 URL에 대해 아래 순서대로 판정이 수행됩니다. 앞 단계에서 확정되면 이후 단계는 건너뜁니다.
1
이미 ①-① 차단완료된 URL
소스에 차단 처리가 완료된 URL — 상태 변경 없이 건너뜁니다.
↓ 아니면
2
담당자가 상태를 확정한 URL
확정 처리된 URL — 기존 상태를 유지하며 자동 재판정에서 제외됩니다.
↓ 아니면
3
현업 검토 결과 '차단 제외'로 확정된 URL
'차단대상 제외'로 검토 완료된 URL — 사용 상태로 유지됩니다.
↓ 아니면 → 자동 판정 실행
🔴 ①-① 차단완료
메서드 본문에 UnsupportedOperationException throw + 메시지 "차단" 포함 (위치 무관).
📝 차단처리미흡 · @Deprecated / [URL차단작업] 주석 중 일부 누락 시 부가 표기
🟠 ①-②/①-③ 최우선 차단대상
실제 사용 이력 없음 + 1년 이상 미수정
🟡 ②-①~②-④ 검토필요대상
사용 이력 있음 + 1년↑ 미수정
또는 사용 없음 + 1년 미만
🟢 사용
위 조건에 해당하지 않는 기본 상태. 정상 운영 중.
🛠️ 개발자 참고용
