ApiViewer 업무 플로우 개요
ApiViewer는 Spring Boot Controller 소스에서 URL을 자동 추출하고, APM 호출건수와 Git 커밋 이력을 결합해 차단 대상 여부를 판정하는 내부 도구입니다. 아래 플로우는 관리자가 소스를 추출·설정하고, 시스템 배치가 주기적으로 데이터를 수집하며, 상태 판정 로직이 자동 계산한 결과를 현업 검토자가 최종 확인하는 흐름을 보여줍니다.
관리자 자동 배치 상태 판정 사용자·검토

3. 상태 판정 로직 (5가지 상태)
RULE · DECISION 상태 판정

@Deprecated + [URL차단작업] 주석 + UnsupportedOperationException
3가지 조건 모두 AND. 모든 수정 불가.
자동 · 변경불가
최우선 차단대상 최우선
호출 0건 + 커밋 1년 경과. 가장 시급히 차단을 검토해야 할 대상.
자동
후순위 차단대상 후순위
침해사고 로그·IT담당자 검토건 등 수동 설정 대상. 현업 협의 후 처리.
수동 지정

호출 0건+커밋 1년 미만 또는 호출 1~N건+커밋 1년 경과. 추가 검토 필요.
자동
사용 사용
